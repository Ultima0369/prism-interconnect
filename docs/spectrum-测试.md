# 棱镜协议 · 测试文档

> 入棱镜须知：  
> 光在此折射，数据在此显形。  
> 所有测试皆为窥探真实的仪式，  
> 请保持敬畏，保持沉默。

---

## 一、测试哲学

```
测试不是验证，而是提问。
每个用例都是一个棱镜切片，
折射出系统在不同维度下的样貌。
```

### 核心原则
1. **折射真实** – 不掩盖，不美化
2. **留白观察** – 给异常留出显现空间
3. **边界显形** – 主动寻找光的断裂处

---

## 二、测试结构

### 2.1 单元棱镜
```python
# 示例：数据折射测试
def test_data_refraction():
    input = pure_light()
    output = prism.refract(input)
    
    # 不断言“正确”，只观察“存在”
    assert output is not None
    assert hasattr(output, 'spectrum')
    
    # 留白：不验证具体波长值
    # 让系统在静默中展现本性
```

### 2.2 集成棱镜
```
测试场景：多棱镜串联
关注点：折射损耗、色散累积、临界角
留白处：不预设光路，记录实际路径
```

### 2.3 混沌棱镜
```
注入噪声光：
- 非结构化数据流
- 极端波长冲击
- 时序错乱

观察系统如何：
1. 折射 → 解析
2. 散射 → 降级
3. 吸收 → 容错
```

## 三、测试仪式

### 3.1 准备阶段
```yaml
环境要求：
  - 暗室条件：隔离外部数据干扰
  - 纯净光源：基准测试数据集
  - 记录介质：日志与指标收集器

精神准备：
  - 清空预设预期
  - 接受任何折射结果
  - 准备见证系统本貌
```

### 3.2 执行阶段
```
步骤：
1. 发射基准光
2. 记录折射模式
3. 标记异常色散
4. 绘制光路图谱

禁忌：
- 不中断进行中的折射
- 不因结果“丑陋”而停止
- 不在同一棱镜上重复照射
```

### 3.3 解读阶段
```
阅读测试报告时：
1. 先看留白处发生了什么
2. 再看断言处遗漏了什么
3. 最后看光路中隐藏了什么

记住：
失败不是错误，
是棱镜在展示它的另一面。
```
