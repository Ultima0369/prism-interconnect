# 🤝 棱镜协议社区指南

## 🎯 社区愿景

### 🔥 火堆旁的对话
**我们的愿景**：建立一个温暖、深度、多元的对话社区，让每个人都能在火堆旁找到自己的位置，分享思考，倾听他人，共同成长。

### 🧠 认知的镜子
**我们的使命**：通过棱镜协议，让每个人都能看到自己思考的过程，理解他人的视角，在对话中扩展认知边界。

### 🌈 多元的和谐
**我们的价值观**：
- **尊重多元**：每个视角都有价值
- **深度对话**：超越表面，深入本质
- **共同成长**：在对话中相互启发
- **开放包容**：欢迎所有真诚的参与者

## 🏗️ 社区结构

### 第三步：参与对话
**火堆旁的对话礼仪：**

#### 1. 🎯 开始对话
```markdown
**好的对话开始：**
- "我最近在思考一个问题..."
- "我在使用棱镜协议时发现..."
- "关于设计哲学，我有一个疑问..."
- "我想分享一个使用案例..."

**避免的对话开始：**
- "这个项目有什么用？"（请先阅读文档）
- "给我写个代码"（请先尝试自己解决）
- "这个设计很蠢"（请提供建设性反馈）
```

#### 2. 🧠 深度对话
```markdown
**促进深度对话：**
- 分享你的思考过程
- 询问他人的视角
- 连接不同的观点
- 提出更深的问题

**对话示例：**
用户A: "我觉得工作与生活的平衡关键是时间管理"
用户B: "有趣！我更多从能量管理角度思考。你能多说点时间管理的具体方法吗？"
用户C: "这两种视角让我想到，也许平衡是动态的，不同阶段需要不同策略..."
```

#### 3. ⏸️ 留白时刻
```markdown
**在对话中留白：**
- 在激烈讨论时暂停，反思
- 给新想法浮现的时间
- 总结之前的讨论要点
- 提出整合性的问题

**留白提示：**
"让我们暂停一下，回顾刚才的讨论..."
"我需要在思考一下这个观点..."
"基于大家的讨论，我想到一个新的角度..."
```

#### 4. 🌈 综合视角
```markdown
**对话的综合：**
- 识别不同观点的共同点
- 连接看似矛盾的观点
- 提出更全面的理解
- 发现新的探索方向

**综合示例：**
"听了大家的讨论，我发现时间管理和能量管理并不是对立的，而是..."
"从多元视角看，这个问题至少有三个层次..."
"综合来看，我们可能需要一个新的框架来理解..."
```
