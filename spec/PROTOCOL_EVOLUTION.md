# 棱镜互联协议演进说明

本文档描述棱镜互联协议（PIP）的版本演进策略、兼容性保证和未来规划。

## 📋 版本策略

### 版本号格式
`主版本.次版本.修订版本`
- **主版本**：不兼容的重大变更
- **次版本**：向后兼容的功能性新增
- **修订版本**：向后兼容的问题修复

### 当前版本
- **v0.1**：初始版本，基础协议定义
- 状态：**实验性**，可能有不兼容变更

### 稳定版本目标
- **v1.0**：协议稳定，提供长期兼容性保证
- 预计时间：社区验证后6-12个月

## 🔄 v0.1 到 v0.2 的演进计划

### 计划新增功能（v0.2）
以下功能计划在v0.2中引入，**保持向后兼容**：

#### 1. 增强的消息元数据
```json
{
  "metadata": {
    "recursion_limit": 3,                    // 递归深度限制
    "context_window": "personal",           // 上下文窗口类型
    "spectrum_weights": {                   // 光谱权重（可选）
      "red": 0.3,
      "blue": 0.4,
      "purple": 0.3
    },
    "preferred_languages": ["zh-CN", "en"], // 语言偏好
    "accessibility_features": {             // 无障碍特性
      "text_complexity": "medium",
      "requires_alt_text": false
    }
  }
}
```

#### 2. 扩展的光谱类型注册
```json
{
  "sender": {
    "id": "custom_agent",
    "capabilities": ["red", "blue", "purple", "green", "orange"],
    "custom_spectra": {
      "green": {
        "name": "生态视角",
        "description": "关注系统关系、循环、长期平衡"
      },
      "orange": {
        "name": "历史经验",
        "description": "基于历史模式、文化传承、集体记忆"
      }
    }
  }
}
```

#### 3. 会话管理扩展
- **会话ID**：跨消息的会话标识
- **消息引用**：明确引用先前消息的机制
- **状态同步**：参与方状态同步协议

#### 4. 性能优化字段
- **estimated_processing_time**：预计处理时间
- **priority**：消息优先级
- **cache_hints**：缓存提示

### 可能的不兼容变更（v1.0考虑）
以下变更**不会在v0.x中引入**，仅作为v1.0的考虑：

1. **必填字段变更**：可能增加新的必填字段
2. **枚举值扩展**：现有枚举字段可能增加新值
3. **格式优化**：JSON结构可能优化以提高可读性
4. **错误处理标准化**：统一的错误响应格式

## ⚠️ 兼容性保证

### v0.x 系列的兼容性
- **v0.1 → v0.2**：完全向后兼容
- **v0.2 → v0.3**：计划保持向后兼容
- **v0.x 内部**：不保证向前兼容（v0.2实现可能不理解v0.3的扩展字段）

### 实现者建议
1. **宽松解析**：解析时忽略未知字段
2. **默认值处理**：为可选字段提供合理默认值
3. **版本检查**：检查`version`字段并适当处理

```python
def handle_message(message):
    version = message.get("version", "0.1")
    
    if version.startswith("0."):
        # v0.x系列，使用宽松解析
        return handle_v0_message(message)
    elif version.startswith("1."):
        # v1.x系列，严格验证
        return handle_v1_message(message)
    else:
        raise ValueError(f"不支持的协议版本: {version}")
```

### 弃用策略
当某个功能被标记为弃用时：
1. **至少保留两个次版本**的兼容性
2. **在文档中明确标记**为弃用
3. **提供迁移指南**
4. **在协议元数据中警告**

## 🤔 参与演进决策

### 决策流程
1. **问题识别**：在GitHub Issues中提出需求或问题
2. **方案讨论**：社区讨论可能的解决方案
3. **原型实现**：创建概念验证实现
4. **反馈收集**：收集用户和开发者反馈
5. **正式采纳**：基于反馈完善并纳入协议

### 决策原则
1. **用户价值优先**：变更必须解决真实问题
2. **简单性保持**：避免不必要的复杂性
3. **向后兼容**：尽可能保持兼容性
4. **渐进演进**：小步快跑，频繁发布
5. **社区共识**：重要变更需要社区讨论

### 如何参与
1. **提出建议**：在GitHub Issues中描述你的想法
2. **参与讨论**：评论现有提案
3. **贡献代码**：实现原型或测试
4. **分享用例**：提供真实使用场景
